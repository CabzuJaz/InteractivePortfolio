Security Practices

Last updated: June 27, 2026

1. Overview

Build with Jazz, operated by Jazzmin Sicat-Cabizares, takes reasonable measures to protect the data we process, including data accessed through Meta platforms such as Facebook Login, Messenger, and the Graph API. This page describes the security practices we apply. No system is perfectly secure, and we continue to improve our practices over time.

2. Encryption in transit (HTTPS)

  • All traffic to our websites and services is served over HTTPS using TLS.
  • Requests to and from Meta APIs are made over encrypted HTTPS connections.
  • We rely on the platform and infrastructure providers we use to maintain current TLS configurations and certificates.

3. API security

  • Access tokens and app secrets are stored as environment secrets, never committed to source control or exposed to the client.
  • We request only the minimum Graph API permissions needed for a feature, consistent with the principle of least privilege.
  • Tokens are transmitted only over HTTPS and are used server-side for calls to Meta APIs.
  • We validate and sanitize input to our endpoints to reduce the risk of injection and abuse.

4. Webhook verification

  • During webhook setup, we verify Meta's subscription request using the configured verify token before accepting the subscription.
  • For incoming webhook events, we validate the X-Hub-Signature-256 header by computing an HMAC-SHA256 of the raw request body using our app secret, and reject any payload whose signature does not match.
  • Webhook endpoints are served only over HTTPS.

5. Access control

  • Access to systems and stored data is limited to authorized personnel only.
  • As an independent developer, access is restricted to the operator of Build with Jazz and the trusted infrastructure providers required to run the Services.
  • We use strong, unique credentials and enable multi-factor authentication on accounts that support it, including our Meta developer account.
  • Secrets and credentials are rotated when there is any indication of compromise.

6. Data minimization and retention

We collect only the data needed to operate the Services and retain it only as long as necessary. For details on what we collect, how it is used, and how to request deletion, see our Privacy Policy and Data Deletion Instructions.

7. Incident response

If we become aware of a security incident affecting personal data, we will investigate, take steps to contain and remediate it, and notify affected users and authorities where required by applicable law.

8. Reporting a vulnerability

If you believe you have found a security vulnerability in our Services, please report it to us at jazzmincabizares@gmail.com. We appreciate responsible disclosure and will work to address valid reports promptly.
