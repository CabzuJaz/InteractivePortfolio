App Description for Meta Review

Application: Build with Jazz — Operated by Jazzmin Sicat-Cabizares — Last updated: June 27, 2026

1. Application overview

Build with Jazz is an AI-powered portfolio and developer showcase operated by an independent software developer. The application allows visitors to interact with an AI assistant that answers questions about the developer's work, skills, and services in a conversational interface.

The Meta platform integration enables the AI assistant to receive and respond to messages sent via Facebook Messenger, and allows users to authenticate using Facebook Login to access personalized features. The application uses the Graph API only to read data explicitly authorized by the user during the login flow.

The application does not publish content on behalf of users, does not access friends' data, and does not engage in advertising, data brokering, or any commercial use of Meta user data beyond operating the Services described here.

2. User flow

2a. Facebook Login flow

  1. The user visits buildwithjazz.com and selects "Continue with Facebook."
  2. The user is redirected to the standard Facebook OAuth dialog, where they review and approve the requested permissions.
  3. On approval, the user is redirected back to the application. The application receives an access token scoped to the approved permissions.
  4. The application reads only the data covered by the granted permissions (e.g., name, email) and uses it to personalize the experience. No data is stored beyond what is necessary for the session or the stated purpose.
  5. The user may revoke access at any time via Facebook Settings → Apps and Websites, or by submitting a data deletion request as described on the Data Deletion page.

2b. Messenger flow

  1. A user sends a message to the Build with Jazz Facebook Page via Messenger.
  2. Meta delivers the message to the application's registered webhook endpoint over HTTPS.
  3. The application validates the X-Hub-Signature-256 header to confirm the payload originated from Meta.
  4. The AI assistant processes the message and sends a reply back to the user via the Messenger Send API.
  5. The conversation is used only to generate the response. Message content is not stored long-term, sold, or used to target advertising.

3. Requested permissions

Permission Reason requested Data used
public_profile Authenticate the user and display their name in the personalized chat interface. Name, profile picture
email Associate the account with a contact email for support and data deletion requests. Email address
pages_messaging Send and receive messages on behalf of the Facebook Page via the Messenger Send API so the AI assistant can reply to users. Messenger message content
pages_show_list Identify the connected Facebook Page in order to configure the Messenger webhook subscription. Page ID, Page name

No additional permissions are requested. All data is used solely for the purposes listed above and is handled in accordance with the Privacy Policy.

4. Webhook usage

The application subscribes to the following Messenger webhook fields on the connected Facebook Page:

  • messages — to receive inbound messages from users so the AI assistant can process and respond to them.
  • messaging_postbacks — to receive button and quick-reply postbacks triggered by interactive message elements.

All webhook events are delivered to a single HTTPS endpoint. The application verifies every incoming payload using HMAC-SHA256 against the app secret before processing. Events not matching the expected signature are rejected with a 403 response. No other webhook fields are subscribed to.

5. Messenger usage

The application uses Messenger solely to operate a user-initiated AI assistant conversation on the Build with Jazz Facebook Page. Specifically:

  • Responses are sent only to users who have initiated a conversation with the Page. The application does not send unsolicited messages.
  • Message content is used only to generate the AI response within the same session. It is not used for profiling, advertising, or any secondary purpose.
  • The application complies with the Messenger Platform Policy, including the 24-hour messaging window for standard messages.

6. Reviewer testing steps

The following credentials and steps are provided to allow the Meta App Review team to verify each permission and feature in the application.

Test account credentials

A dedicated test account has been provided in the App Review submission form.

If credentials were not included there, please contact jazzmincabizares@gmail.com and they will be provided within one business day.

Testing Facebook Login (public_profile, email)

  1. Navigate to https://buildwithjazz.com.
  2. Select "Continue with Facebook".
  3. Log in using the provided test account and approve the requested permissions in the Facebook OAuth dialog.
  4. Confirm that you are redirected back to the site and that the personalized greeting (name from public_profile) appears in the chat interface.
  5. Confirm that no data beyond name, profile picture, and email is accessed or displayed.

Testing Messenger (pages_messaging)

  1. Using the provided test account, open Messenger and search for the Build with Jazz Page, or navigate directly to the Page and select "Send Message."
  2. Send any message, for example: "Hi, what services do you offer?"
  3. Confirm that the AI assistant replies within a few seconds with a relevant response.
  4. Confirm that no content is sent by the application unless you initiate the conversation (no unsolicited messages).

Testing Page access (pages_show_list)

  1. Log in using the provided test account (which has a Page role on the Build with Jazz Page).
  2. The application uses pages_show_list only to identify the connected Page during initial setup. There is no user-facing feature that displays Page list data.
  3. Confirm that the Messenger webhook subscription is active by sending a Messenger message as described in the Messenger test steps above.

Data deletion verification

  1. Navigate to https://buildwithjazz.com/legal/data-deletion.html.
  2. Confirm that clear instructions for requesting data deletion are present, including an email contact and the Facebook Apps and Websites removal path.
